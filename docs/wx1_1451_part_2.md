## 2.2 Accessing the xAPI via SSH
In this section, we'll dive into the various pieces of the xAPI stack and how to make use of them in various ways over an SSH Session to the codec.

Topics covered for SSH nearly 1:1 match a use case driven via a Serial Connection, whereas Serial requires additional hardware, it will not be covered in depth in this Lab.

### 2.2.1 - Establish SSH Connection to Device

- Open Terminal
- Connect to the Device via SSH
  - Type `ssh [USERNAME]@[IP_ADDRESS]`, then hit enter
    - Example: `ssh admin@10.0.0.X`
  - Type `Y` or `Yes` when prompted, then hit enter
  - Type the admin account password when prompted, then hit enter

### 2.2.2 - Navigating the Terminal

- Change Output Preferences
  - `xpref outputmode ?`
- List Command References
  - `xCommand ?`
- List Status Nodes
  - `xStatus ?`
- List Config Nodes
  - `xConfiguration ?`
- List Event Nodes
  - `xEvent ?`
- Search for an xAPI using a Wildcard
  - `xConfig // Name ?`

### 2.2.4 - Getting, Setting and Subscribing to Configurations
	A. Get a Config Value
		a. xConifg Audio DefaultVolume
	B. Set a new Config Value
		a. xConfiguration Audio DefaultVolume: 40
	C. Get Multiple Config Values under a common Node
		a. xConfig Audio
	D. Subscribe to a Config
		a. xFeedback Register Configuration/Audio/DefaultVolume
	E. Unsubscribe from a Config
		a. xFeedback Deregister Configuration/Audio/DefaultVolume
	F. Subscribe to multiple Config under a common node
		a. xFeedback Register Configuration/Video/Input/AirPlay
	G. Unsubscribe from all
		a. xFeedback DeregisterAll

### 2.2.5 - Setting and Subscribing to Status
	A. Get a status value
		a. xStatus Audio Volume
	B. Get multiple Status Values under a common node
		a. xStatus Audio Input
	C. Subscribe to a Status
		a. xFeedback Register Status/Audio/Volume
	D. Unsubscribe from a Status
		a. xFeedback Deregister Status/Audio/Volume
	E. Subscribe to multiple Statuses under a common node
		a. xFeedback Register Status/RoomAnalytics
	F. Unsubscribe from all
		a. xFeedback DeregisterAll
